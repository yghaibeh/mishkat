## الحالة: جاهز للنشر ✅
- البناء ينجح، و`wrangler deploy --dry-run` يبني الـWorker بنجاح (≈495 KiB مضغوط) ويتعرّف على الربطين `DB` (D1) و`ASSETS`.
- المخطط مكتمل في `migrations/` (0000…0009، يشمل `permission_overrides` واللجان).
- سكربتات النشر مضافة في `package.json`.
- الأسرار مؤمّنة: `.dev.vars` محليًّا، و`[vars]` في wrangler.toml **خالٍ** عمدًا.

## المتطلّبات
- حساب Cloudflare (الطبقة المجانية تكفي للـPilot).
- Node + npm، و`wrangler` عبر `npx` (مثبّت، 4.86).
- العمل من مجلّد `app/`.

## الخطوات

### ١) تسجيل الدخول (تفاعلي — يفتح المتصفّح)
```bash
cd app
npx wrangler login
```

### ٢) إنشاء قاعدة D1 ووضع المعرّف
```bash
npx wrangler d1 create influential-masjid
```
انسخ `database_id` من المخرجات وضعه في `app/wrangler.toml` مكان `00000000-0000-0000-0000-000000000000`.

### ٣) سرّ توقيع الجلسات JWT (لا يوضع في أي ملف)
```bash
openssl rand -base64 48        # ولّد سرًّا قويًّا
npx wrangler secret put JWT_SECRET   # الصق القيمة عند الطلب
```

### ٤) المخطط (migrations) على القاعدة البعيدة
```bash
npm run cf:migrate:remote      # يطبّق 0000…0009
```

### ٥) البيانات الأوّلية (Pilot إدلب)
```bash
npm run cf:seed:remote
```
يطبّق بالترتيب: `seed_points` (المعدّلات + جداول التنقيط) ← `seed_demo` (إدلب: الهيكل + `admin`/`amir` + تقارير) ← `seed_users` (بقية الأدوار: المشرفون… مثل `square2`).
> `seed_big.sql` بيانات اصطناعية ضخمة **للتفحّص المحلي فقط** — لا تُرفع للـPilot. وحدات «على بصيرة/اللجان/الاجتماعات/التحفيظ» تبدأ فارغة ويُدخلها فريق إدلب.

### ٦) النشر
```bash
npm run deploy                 # vite build ثم wrangler deploy
```

### ٧) التحقّق بعد النشر
- افتح رابط الـWorker، وسجّل الدخول بـ `admin` (كلمة المرور المبذورة: `mishkat123`).
- **بدّل كلمات المرور المبذورة فورًا** (admin والحسابات الأخرى) من /admin ← المستخدمون.
- اضبط المعدّلات المالية من /admin ← المعدّلات إن لزم.

## الأمان (مطبّق مسبقًا)
- `.dev.vars` و`.wrangler-state/` مستثناة في `.gitignore`.
- `JWT_SECRET` **سرّ Cloudflare** لا نصّ ظاهر في wrangler.toml (لئلّا تُزوَّر الجلسات).
- كلمات المرور المبذورة معروفة عمومًا — **يجب** تبديلها بعد أوّل دخول.

## استكشاف الأعطال
| العرض | السبب الغالب | الحل |
|---|---|---|
| الأصول 404 | لم يُبنَ العميل | `npm run build` قبل `deploy` (موجود ضمن `deploy`) |
| خطأ قاعدة بيانات | `database_id` وهمي أو لم تُطبَّق migrations | ضع المعرّف الحقيقي + `npm run cf:migrate:remote` |
| 500 عند الدخول | `JWT_SECRET` غير مضبوط | `npm run cf:secret` |
| `wrangler login` يفشل | بيئة بلا متصفّح | استخدم `CLOUDFLARE_API_TOKEN` بدل `login` |

## التحديثات اللاحقة
- كود فقط: `npm run deploy`.
- مخطط جديد: أضف ملف migration ثم `npm run cf:migrate:remote` ثم `npm run deploy`.
