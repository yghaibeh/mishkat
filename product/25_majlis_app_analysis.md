# 25 — تحليلٌ شاملٌ لتطبيق «مجلس» (from_client) — أفكاره وميزاته، وكيف نستفيد

> استُخرج بتفكيك `from_client/app-release-2.apk` (تطبيق Flutter، حزمة `com.example.majlis`، الاسم «تطبيق مجلس»). المنطق (Dart) في `libapp.so`؛ استُخرجت منه ٢٨٢ عبارة واجهة عربية + معرّفات جداول قاعدة البيانات + أسماء الشاشات. الصلاحيات: **الكاميرا** (مسح QR) + الإنترنت.
> **الخلفيّة الحقيقيّة: Supabase** (Postgres + Auth + Realtime + Edge Functions + Storage) على المشروع `ixiadmkgjuvylsqixgrt.supabase.co`، و**الرسائل النصّية عبر `tweetsms.ps`** (بوّابة SMS فلسطينيّة — تؤكّد الجهة الغزّيّة). (مكتبات Firebase مُجمَّعةٌ في الحزمة لكنّها ليست الخلفيّة العاملة.) ⇐ **النموذج علائقيّ (Postgres)**، أقربُ مباشرةً لمعمارية مِشكاة (D1/SQLite).

## 0) الهُويّة والتقنية
- **الاسم:** «مجلس» — تطبيقٌ لإدارة **المجالس/الحلقات العلمية** (طلب علمٍ شرعيّ: عقيدة، فقه، علوم القرآن، تفسير، متون كبلوغ المرام).
- **التقنية:** Flutter (Dart AOT) + **Supabase** (Postgres/Auth/Realtime/Edge Functions/Storage) + **SMS عبر tweetsms.ps** + مسح QR (ML Kit) + auto_route (تنقّل).
- **الجمهور:** متعدّد الدول (السعودية، العراق، الأردن، اليمن، سوريا، ليبيا، تونس، تركيا، ماليزيا، إندونيسيا، المغرب) مع تمييزٍ خاصّ **«داخل غزة / خارج غزة»** (شاشتا تسجيلٍ منفصلتان) — مؤسّسةٌ دعويّة غزّيّة على الأرجح.

## 1) الأدوار (Actors) — خمسة
| الدور (الواجهة) | في الكود | الصلاحية |
|---|---|---|
| **الشيخ** | `TeacherShellScreen` | الأعلى: يدير المواد والدروس والاختبارات والخطب والنشاطات، ويضيف المشرفين |
| **المشرف** | `SupervisorShellScreen` | نطاقٌ إشرافيّ (`نطاق الإشراف`): مثل الشيخ **+ قسم الطلاب** — يضيف طالبًا، يسجّل الحضور، يتابع |
| **الطالب** | `StudentShellScreen` | يتابع المطلوب منه، يردّ على النشاطات، يسلّم الواجبات، يرى جدوله وحضوره |
| **السائق** | (ضمن الإدارة) | لوجستيّات النقل (`إضافة/حذف السائق`) |
| **الإدارة** | `ManagementScreen` | إدارةٌ عليا للمنصّة + إعلانات |

- **تعدّد الأدوار:** «**ترقية الطالب لمشرف**» + «المشرف نفسه طالب» ⇒ الشخص الواحد يحمل دورَين (يطابق رسالة أصحاب المشروع: مشرف=طالب، الكود 470002).
- **بوّابةٌ بالدور:** رسائل «هذا الحساب غير مسجّل كشيخ/كطالب/كمشرف» ⇒ الدخول يتحقّق من دور الحساب.

## 2) المصادقة والتسجيل
- **الدخول بالكود + كلمة المرور:** كلّ مستخدمٍ له **كودٌ رقميّ** (`الكود`/`رقم الكود`/`كود الدخول`)، مثل `470001`. «تم إرسال كود الدخول إلى رقم جوالك، استخدمه مع كلمة المرور» ⇒ الكود يُرسَل عبر الجوال.
- **OTP:** `OtpVerifyScreen` + «رمز التحقق» — تحقّقٌ ثنائيّ.
- **التسجيل حسب المنطقة:** `GazaRegisterScreen` / `OutsideRegisterScreen` / `RegisterRegionScreen` — «اختر مكان إقامتك للتسجيل»؛ حقول: الاسم الرباعي · رقم الهوية · تاريخ الميلاد · الجنس · رقم الجوال/الواتساب · البريد · الدولة.
- استعادة/تغيير كلمة المرور (`ForgotPasswordScreen`/`ChangePasswordScreen`/`SetPasswordScreen`)؛ الحدّ ٨ خانات.
- مسح **QR** لتسجيل الدخول/الحضور السريع (`مسح QR` + صلاحية الكاميرا).

## 3) نموذج البيانات (جداول Supabase/Postgres المستخرَجة)
`users / profiles` · `sheikhs` · `supervisors` · `students` · `drivers` · `subjects` (المواد) · `categories` (التصنيفات) · `materials` (الملفّات العلمية) · `lessons` · `lectures` (المحاضرات) · `sessions` (الجلسات) · `schedules` · `circles` (الحلقات/المجالس) · `attendance` (الحضور) · `activities` (النشاطات) · `responses` (ردود الطلاب) · `tasks` · `exams/quizzes` + `questions` + submissions · `sermons` (الخطب) · `custody` (العُهد) · `notifications` · `announcements`.
- **حقولٌ ملحوظة:** `role · code · required/isRequired · dueDate · repeat (تكرار) · options · type · country · gender · nationalId · birthDate · createdAt · updatedAt · expiresAt · audioUrl/pdfUrl · points/score`.

## 4) وحدات الميزات (كلّ فكرة)

### أ) المواد العلمية والتصنيفات (`MaterialsScreen`)
- مادّةٌ علميّة لها **تصنيف** (عقيدة · فقه · علوم القرآن · تفسير …) و**ملفّات** مرفقة (صوت/PDF): «إرفاق ملف صوتي أو PDF» · «تحميل PDF». «تصنيفاتك وموادّك العلمية».
- حذف المادة يحذف دروسها واختباراتها وخطبها المرتبطة (كيانٌ جامع).

### ب) الدروس والمحاضرات والجدولة (`LessonDetailScreen`/`LectureDetailScreen`)
- درس/محاضرة: عنوان · وصف · **مكان** · **تاريخ/موعد** · **مدّة بالدقائق** · مادّة مرتبطة (مثال: «تفسير سورة الكهف»، «إلى حلقة بلوغ المرام»).
- **كشف تعارض المواعيد:** «يوجد درسٌ آخر في نفس الموعد».
- **جلسات اليوم** (`TodaySessionTile`) · **الجدول القادم** · حالة الدرس (`LessonStatusSheet`: قادمة/مفتوح/مؤكّد…).
- «الدروس المُلقاة» + طباعة (`PrintScreen`).

### ج) الحضور (`AddAttendeeSheet`)
- سجلّ حضورٍ لكلّ درس: «إضافة طالب للحضور» · «انضمام الطلاب وغيابهم» · «حضورك في المواد» (تقرير حضور الطالب عبر المواد).

### د) النشاطات والمتابعة — ✦ جوهر الفكرة (`StudentActivitiesScreen`/`FollowUpScreen`/`AddActivitySheet`)
- الشيخ/المشرف يُنشئ **نشاطًا مطلوبًا** من الطلاب (`required`)، والطالب **يردّ** عليه (`responses`): «**نشاطاتك المطلوبة وردودك**» · «تم إرسال ردّك» · «تم الرد».
- **`FollowUpScreen`** = شاشة المتابعة: يرى الطالب ما هو **مطلوبٌ منه** بسلاسة، ويرى الشيخ ردود طلابه. (هذا ما قصده أصحاب المشروع: متابعةٌ سلسة للمطلوب.)

### هـ) الاختبارات والواجبات (`ExamDetailScreen`/`AddExamSheet`/`AddQuestionSheet`)
- «أنشئ اختبارًا جديدًا لطلابك» · أسئلة بنوع **اختيار من متعدّد** أو **صح/خطأ** · «نص السؤال».
- **نشر** الاختبار · **وقت تسليم** («انتهى وقت التسليم») · «تسليم واجب» · درجات (`points/score`) · «تحديثات الواجبات والاختبارات».

### و) الخطب (`AddSermonSheet`/`SermonDetailsSheet`)
- خطب الجمعة: عنوان + **ملف صوتيّ** (مثال: «خطبة عن بر الوالدين») · «إجمالي الخطب».

### ز) العُهد (`AddCustodySheet`)
- تتبّع **العُهدة/الأصول** (مثل «لابتوب»): «اختر نوع العهدة» · «لا توجد عُهد مسجّلة» — مَن بحوزته أيّ أصل.

### ح) السائقون/النقل (`AddDriverSheet`)
- إدارة سائقي النقل: «إضافة/حذف السائق» · «لا يوجد سائقون بعد».

### ط) التنبيهات والتذكير — ✦ «برنامجٌ تذكيريّ» (`NotificationsScreen`/`NotificationSettingsScreen`)
- **«تذكير بحلقة اليوم»** · **«تنبيهات النشاطات»** · «تحديثات الواجبات والاختبارات» · «تذكّرني» · تذكيراتٌ نسبيّة («بعد ٣ أيام»، «قبل ٣ ساعات»).
- **إعلانات وأخبار المنصّة** · إعدادات إشعاراتٍ دقيقة.

### ي) عامّ
- الملف الشخصيّ (`ProfileScreen`) · سِمة داكن/فاتح · بحثٌ بالاسم أو الكود · طباعة التقارير · ملاحظات الشيخ.

## 5) خريطة الشاشات حسب الدور
- **الشيخ (Teacher):** لوحة · مواده وتصنيفاته · دروس/محاضرات · اختبارات + أسئلة · خطب · نشاطات + ردود · حضور · طباعة.
- **المشرف (Supervisor):** كالشيخ **+ الطلاب** (`StudentsScreen`): إضافة طالب، سجل حضور، متابعة، ترقية طالبٍ لمشرف، نطاق إشراف.
- **الطالب (Student):** لوحة (`StudentDashboardScreen`) · جلسات اليوم · **نشاطاته المطلوبة + ردوده** (`FollowUpScreen`) · حضوره · المواد والاختبارات · الإشعارات.

## 6) الفكرة المحوريّة (كما أكّدها أصحاب المشروع)
> «موضوع الطلاب: للطالب متابعةٌ سلسة لما هو مطلوبٌ منه؛ والشيخ والمشرف يعرف كلٌّ منهما المطلوب منه يوميًّا — فهو برنامجٌ تذكيريّ.»

يتحقّق هذا في «مجلس» عبر ثلاثيّة:
1. **جلسات/جدول اليوم** لكلّ دور (ماذا عليّ اليوم؟).
2. **النشاطات المطلوبة + الردود** (`activities`/`responses` + `FollowUpScreen`) — مهامٌّ مطلوبةٌ يتابعها الطرفان.
3. **إشعاراتٌ تذكيريّة** (حلقة اليوم، النشاطات، الواجبات، الجدول القادم) — تدفع المطلوب في وقته.

## 7) مقارنةٌ بمِشكاة — ماذا نملك وماذا نستفيد
| فكرة «مجلس» | حالها في مِشكاة | فرصة الاستفادة |
|---|---|---|
| كود دخولٍ رقميّ + OTP بالجوال | دخولٌ باسمٍ نصّيّ (لا OTP) | **دخولٌ برمزٍ رقميّ + OTP** أسهل للميدان (يناسب رسالة 470001) |
| المواد العلمية + ملفّات صوت/PDF مصنّفة | مرفقات درسٍ فقط (صور) | **مكتبة موادّ مصنّفة** (عقيدة/فقه…) بملفّات صوت/PDF |
| جدولة الدروس + كشف تعارض المواعيد + جلسات اليوم | تسجيل درسٍ بلا جدولةٍ مسبقة | **جدولٌ مُسبقٌ + «ماذا عليّ اليوم»** لكلّ دور |
| **النشاطات المطلوبة + ردود الطالب + شاشة متابعة** | لا يوجد نظير مباشر | ✦ **الأعلى قيمة**: مهامٌّ مطلوبةٌ للطالب/المعلّم + متابعةٌ سلسة |
| اختبارات (MCQ/صح-خطأ) + واجبات بمواعيد تسليم | لا يوجد | **اختبارات/واجبات** للطلاب |
| خطب الجمعة (صوت) | لا يوجد | وحدة **خطب** للمسجد |
| العُهد (أصول) + السائقون (نقل) | لا يوجد | **عُهد/أصول** ونقل (لوجستيّات) |
| **تذكيراتٌ دفعيّة** (حلقة اليوم/النشاطات/الواجبات) | Web Push + تيليغرام (بُنيا للتوّ) | **نُوجّه قنواتنا القائمة** لتذكيرات «المطلوب اليوم» |
| تسجيلٌ حسب المنطقة (غزة/خارجها) | تسجيلٌ إداريّ فقط | تسجيلٌ ذاتيّ حسب المنطقة (إن لزم) |
| اطّلاع الطالب على متابعته | بنينا أساسه (سجلّ الطالب) | **صفحة الطالب/وليّ الأمر** (المرحلة د) |

## 8) توصياتٌ للنقاش (كيف نستفيد — لاحقًا)
أعلى ثلاث قيمًا للاقتباس إلى مِشكاة، مرتّبةً:
1. **محرّك «المطلوب اليوم» + المتابعة** (نشاطات مطلوبة + ردود + جدول اليوم لكلّ دور) — يوحّد قيمة «مجلس» الجوهريّة مع ما بنيناه (سجلّات/اعتماد)، ويُغذّي قنوات التذكير (Push/تيليغرام) القائمة.
2. **مكتبة الموادّ العلمية المصنّفة** (صوت/PDF) + **جدولة الدروس** (بجلسات اليوم وكشف التعارض).
3. **اختبارات/واجبات الطلاب** + **دخولٌ برمزٍ رقميّ/OTP**.
(العُهد/السائقون/الخطب: قيمةٌ أقلّ إلحاحًا — تُدرَس حسب حاجة بصائر.)

> **الخطوة التالية:** ننتقي معًا ما نقتبسه ونحوّله إلى خطة تنفيذٍ عالميّة (لا Workarounds)، منسجمةً مع معمارية مِشكاة القائمة (الأدوار/النطاق/الاعتماد/الإشعارات).
