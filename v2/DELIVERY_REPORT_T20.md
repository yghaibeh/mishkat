## ١. ما بُني — وما عالجه

المكتبةُ في v1 **كانت تعمل**، وفيها ثغرةٌ واحدةٌ مسجَّلة وثلاثةُ عيوبٍ بنيويةٍ أصغر. وهذه
الوحدةُ تُغلقها **بالبنية لا بالانضباط**:

| العيب في v1 | موضعُه | العلاج في v2 | يُقاس بـ |
|---|---|---|---|
| **ح-٦**: رأسُ قسمٍ يعدّل موادَّ كل الأقسام | `materials.server.ts:118-153` — `canManage` **بلا بُعدٍ نطاقيٍّ إطلاقاً** | نطاقُ كل فعلٍ **يُشتقّ من المادة المخزَّنة** ونوعُه «و» | `scoped-management.test.ts` (٨ حالات) |
| **الجمهورُ قائمةُ أدوارٍ مُصلَّبة** | `const SUPERVISOR_ROLES = [...]` ثم `roles.has(r)` | **الجمهورُ قدرةٌ يُسأل عنها المحرّك** — صفر اسمِ دورٍ في الوحدة | `audience.test.ts` + **G6** |
| **«أنجز» يختم «فتح» صامتاً** | `if (field === "completedAt" && !p.openedAt) patch.openedAt = now` | **يُردّ الطلبُ ويُسمّى الناقص** (`NOT_OPENED_YET`) | `timeline.test.ts` |
| **الموادُّ شبكيةٌ في القراءة** (لا مسارَ وحدةٍ على المادة) | النموذجُ نفسُه | `unitPath` **حقلٌ إلزاميُّ النوع** — فلا موضعَ لمادةٍ بلا موطن | `mine.test.ts` · `tracking.test.ts` |

**والوحدةُ في جملة**: تعرف **مادةً وموطنَها وجمهورَها وإلزامَها** و**ثلاثَ خَتماتٍ لكل شخص**؛
ولا تعرف **مُجدوِلاً** ولا **قناةَ إشعار** ولا **دوراً** ولا **اعتماداً**.

## ٢. **إثباتُ تنطيق `library.manage`** (المطلوب صراحةً في المهمة)

**الآليةُ في سطر**: `library.manage` نوعُ نطاقها **«و» (الاحتواء)** في الملف الذهبيّ،
و**مُحلِّلُ نطاق كل فعلٍ على مادةٍ قائمة يقرأ `unitPath` من المادة المخزَّنة**:

```ts
function materialScope(store, materialId) {
  const material = materialId === undefined ? null : store.getMaterial(materialId)
  return material === null ? NO_SCOPE : unitScope(material.unitPath)   // ← لا شيءَ من المدخل
}
```

**والبرهانُ سلوكيّ لا وعديّ** — رأسُ قسم الشباب (`u-section-head` على `/men/`):

| الفعل | المادة | النتيجة | السبب المُشخِّص |
|---|---|---|---|
| `library.material.update` | في `/women/` | **مرفوض** | `DENIED_OUT_OF_SCOPE` |
| `library.material.archive` | في `/women/` | **مرفوض** | `DENIED_OUT_OF_SCOPE` |
| `library.material.create` | على `/women/` | **مرفوض** | `DENIED_OUT_OF_SCOPE` |
| `library.manage.view` | صفحة `/women/` | **مرفوض** | `DENIED_OUT_OF_SCOPE` |
| `library.material.update` | في `men` · `homs` · `khalid` | **مقبول** (الاحتواء الهابط) | — |
| أيُّ فعلٍ | **مادةٌ مجهولة** | **مرفوض** | `NO_SCOPE` ⇒ يُقفل ولا يُفتح |
| أيُّ فعلٍ | مادةٌ في **شبكةٍ أخرى** بنفس المسار | **مرفوض** | لا تُحلّ أصلاً ⇒ `NO_SCOPE` |

**والإدارةُ على الجذر تبلغ القسمين** — **لأنّ نطاقَ تكليفها الجذرُ صراحةً**، لا لأنها «مدير»
(لا مفهومَ «مدير» في v2 — ت-٥). **ومَن لا يملك القدرةَ مرفوضٌ ولو كان في النطاق**: الأميرُ
والمنطقةُ والمربعُ والمعلّمُ والطالب — و**المنطقةُ محظورةٌ نصّاً** في جدول الغياب §٣.

**والكسرُ يُثبت أنّ الحارسَ حيّ**: حين جُعل النطاقُ ثابتاً بدل أن يُشتقّ من المادة ⇒ **سقط
اختباران** أوّلُهما «رأسُ قسم الشباب يعدّل مادةَ القسم النسائيّ ⇒ مرفوض».

## ٣. **الواجهةُ المعلنة للتذكير** (لمن يستهلكها) — ولا مُجدوِلَ مبنيّ

**لا مُجدوِلَ في v2 بعد**، وبناؤه هنا يُنشئ **الثانيَ من صنفه** (عينُ مرض v1). فالمُصدَّرُ
**الاشتقاق** وحده:

```ts
// v2/src/features/library/services/reminders.ts
export const OVERDUE_SETTING = "materials.mandatory_overdue_days"

export type OverdueMandatory = {
  readonly personId: string            // مَن تأخّر
  readonly materialId: string          // عن أيّ مادة
  readonly materialTitleAr: string     // بعنوانها (فلا يستعلم المستهلِك ثانيةً)
  readonly unitPath: string            // في أيّ وحدة (للتوجيه والعزل)
  readonly deliveredDayKey: string     // متى استلمها (مفتاحُ يومٍ لا لحظة)
  readonly daysSinceDelivered: number  // منذ كم يوم
}

export function overdueMandatory(
  store: LibraryStore, ctx: LibraryContext, scopePath: string,
): readonly OverdueMandatory[]
```

**عقدُها لمن يستهلكها** (وحدةُ الإشعارات/المجدولات حين تُبنى):

1. **تُعيد وقائعَ لا أحكاماً**: «مَن تأخّر ومنذ كم» — **ولا تقول كيف يُبلَّغ ولا متى يُكرَّر**.
2. **العتبةُ إعدادٌ حيّ** `materials.mandatory_overdue_days` تُقرأ **على مسار المادة** — فضبطُها
   على قسمٍ يغيّر سلوكَه فوراً بلا كود. **والتأخّرُ ما جاوزها** (عند الحدّ تماماً لم تنقضِ).
3. **الدوريةُ ليست شأنَ هذه الوحدة**: `materials.reminder_interval_days` **إعدادُ المستهلِك**،
   و**اختبارٌ محتوائيٌّ يفشل إن قرأته هذه الوحدة يوماً**.
4. **مسطَّحةٌ عبر الحدّ كذلك**: `library.overdue.view` بقدرة `library.manage` على النطاق —
   فالمصفوفةُ والإشعارُ **يقرآن الاشتقاق نفسَه**، ولا حسابَ ثانٍ يتباعد.
5. **الترتيبُ حتميّ**: الأطولُ تأخّراً أولاً ثم المعرّفان — قائمةُ عملٍ لا جدولُ أرقام.

> **حدٌّ مُعلَنٌ ومُختبَر لا فجوةٌ صامتة**: التأخّرُ يُقاس **من الاستلام**، والاستلامُ يُختم عند
> أول عرضٍ لمكتبتي (ق-٩٦ نصاً). فمَن **لم يفتح مكتبتَه قطُّ** ليس «متأخراً» بل «لم يستلِم»،
> ولا يظهر في هذه الواجهة — **ولم أخترع له خَتمَ استلامٍ** لأنّ ذلك يجعل خطَّ الزمن كذباً
> (وهو عينُ عطب v1 الذي عالجتُه). وملاحقةُ **غير المستلِمين** سؤالٌ آخر تجيبه مصفوفةُ
> المتابعة (`notDelivered` خليّةٌ ظاهرة)، ووصلُه بقناةِ إبلاغٍ **يمرّ بـCR**.
> **الاختبار الذي يحرس هذا الحدّ موجودٌ ومسمّى** («مَن لم يفتح مكتبتَه قطُّ… لا يُعدّ متأخراً»).

استدعاء الخادم المباشر** (الطبقتان معاً).

## ٨. ثلاثةُ قراراتٍ تصميمية أُعلنها للمدير صراحةً

**١) «الجمهورُ قدرةٌ لا دور» — إعادةُ صوغٍ لا التفاف.**
ق-٩٦ يقول «تُوجَّه لجمهورٍ **بدوره**»، وG6 تمنع أن يعرف كودُ الميزة اسمَ دور. فلم أُخفِ
قائمةَ الأدوار خلف بياناتٍ، بل **غيّرتُ السؤال**: الجمهورُ **حزمةُ عملٍ**، وحزمةُ العمل في
v2 اسمُها **قدرة**. والصفُّ المرجعيُّ يحمل `capabilityId`، والانتماءُ **جوابُ المحرّك**
(شخصيةٌ ⇐ بملكية الشخص نفسِه — نصُّ قب-٣٨؛ منطاقةٌ ⇐ على مسارات تكليفه).
**والمطابقةُ مع v1 تامّةٌ ومُختبَرة**: `visit.conduct` يحملها **مشرفو المربع والمنطقة والقسم
حصراً** — وهي `SUPERVISOR_ROLES` نفسُها بلا قائمة. **والمكسب**: دورٌ جديدٌ يدخل جمهورَه يومَ
تُمنح له القدرةُ في الملف الذهبيّ — **بلا سطرِ كودٍ وبلا قائمةٍ يتذكّرها إنسان**.
*(النمط مسبوق: `makePublishingScopeCheck` في الإعلام تسأل المحرّك بقدرةٍ لا بدور.)*

**٢) «مكتبتي» تُعلن نيّتَها كتابةً.**
ق-٩٦ يوجب ختمَ الاستلام عند أول عرض ⇒ **العرضُ فعلٌ كاتب**. فأعلنتُ `library.mine.view`
بـ`intent: "write"` — إعلانٌ يصف **الفعل** لا **الشاشة**. وثمرتُه مُختبَرة: **الانتحالُ
القرائيّ (ب-٤٠أ) لا يختم باسم أحد** ⇒ `DENIED_IMPERSONATION_READONLY`.

**٣) بلوغُ المادة = جمهورٌ **و**نطاق.**
v1 لم يكن للمادة نطاقٌ أصلاً (وهو أصلُ ح-٦). وفي v2 المادةُ تبلغ مَن **في** وحدتها هابطاً:
فمادةُ الجذر تبلغ الشبكة، ومادةُ `/men/` تبلغ مَن تحته، ومادةُ مسجدٍ تبلغ أهله.
**والمديرُ على الجذر لا تبلغه مادةُ قسمٍ** — لأنه ليس **من أهله**، ومن أراد الشمولَ أنشأها
على الجذر. **هذا تشديدٌ على v1 (لا تخفيف)** وهو ما تقتضيه ق-١٧، ومُختبَرٌ سلباً وإيجاباً.

## ٩. ملاحظةٌ لمدير البرنامج — **G15 حمراءُ على `main` قبل تسليمي** (لا بسببه)

شغّلتُ البواباتِ على `main` **بلا أيّ تغييرٍ من عندي** فخرجت **G15 حمراء**:

```
✗ G15 — مرجع طلب التغيير
   • تسليم جوهري بلا طلب تغيير
   •   rebuild/specs/SPEC_settings.md
   •   v2/src/settings/registry.ts
```

**السبب**: التزامُ T17 الأخير (`9b267f5`) مسّ سجلَّ الإعدادات ومواصفتَه (وهو **تنفيذُ CR-014
المعتمد**)، **ورسالتُه لا تحمل `CR-014` نصّاً** — والبوابةُ تقرأ الرسالة. فالخللُ **في الرسالة
لا في العمل**.

- **لم أمسّه** (رسالةُ التزامٍ مدموج ملكُ المدير لا الوكيل — والتعديلُ يعيد كتابة التاريخ).
- **وبعد التزامي خرجت G15 خضراء** («لا يمسّ أصلاً حسّاساً») — فتسليمي **لا يخفي** الحمرةَ
  السابقة ولا يُبقيها في مسار البناء.
- **والدرسُ المقترح** (بصيغة قب-): *بوابةٌ تحرس **رسالةَ** الالتزام تُفحص **قبل** الدمج لا
  بعده — فبعد الدمج يصير علاجُها إعادةَ كتابة تاريخ. تُضاف خطوةُ «شغّل G15 على الرسالة قبل
  الدمج» إلى مسؤوليات المدير (§٦ من بروتوكول التوازي).*

**وتنبيهُ دمجٍ متوقَّع** (قب-٣٧): نقاطُ التسجيل الثلاث ستتصادم مع T18/T19/T21 الجارية —
**تُدمج يدوياً وتُتحقَّق بالبوابات بعد كل دمج**، ولا ثقةَ بالضمّ الآليّ.
